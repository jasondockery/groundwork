AI-native Mac development

An AI-native Mac dev environment you can rebuild from scratch.

Every app, config, keybinding, project instruction, and agent workflow on this machine comes from one small repository. Groundwork teaches the machine setup and the human workflow: how to use the terminal, Git, editors, and coding agents together without losing control of the work.

Quick start

Want a configured Mac now? Run the bootstrap. Want to understand what it did? Keep reading from Getting started, Setup & chezmoi, and Customizing.

bootstrap
$ bash -c "$(curl -fsSL https://raw.githubusercontent.com/jasondockery/groundwork/main/bootstrap-mac.sh)"

#How it's built

Two tools do the heavy lifting. Homebrew installs the software. chezmoi takes a folder of config files kept in version control and lays them down in the right places in your home directory — so a brand-new Mac becomes a fully configured one with a single command, and every machine stays identical.

You don't have to trust that blindly. The Setup & chezmoi page walks through exactly what chezmoi apply does and why you run it, so you understand the system rather than memorizing commands.

The idea

Your configuration is code, and your agent instructions are code-adjacent operating rules. Both live in Git, both are reviewed and versioned, and both can be reproduced on any Mac. Break something? Re-run one command and you're back.

#Why terminal-first still matters

AI-native does not mean the terminal goes away. It means the terminal becomes the control room: agents read files, run shell commands, inspect Git diffs, apply project instructions, and verify changes. tmux keeps that work visible, Neovim and VS Code let you intervene by hand, and Git gives every experiment a safe checkpoint.

The direction is intentionally written down in one place: AI_THESIS.md. Agent adapters, skills, and docs reference that north star instead of each keeping their own copy.

#A path to follow

Don't try to learn everything at once. Pick one thing, practice it, move on. This order works well:

1

Open the terminal

If the command line is new to you, start with Getting started — what a terminal, a prompt, and a file path are, and how to open Ghostty. Fifteen minutes, and everything below makes sense.

2

Learn the command line

Work through Command line & Unix — core commands, piping, a first script, and what Unix actually is. The ground the other tools stand on.

3

Learn to type in Vim

Run vimtutor in your terminal. It's a free, hands-on, 30-minute lesson built into the Mac. Do it twice over two days.

4

Understand the shell

Work through The Missing Semester — MIT's free course on the command line, Vim, tmux, and Git. The single best foundation.

5

Get comfortable in tmux

Read the tmux page and run its practice drills. Splitting panes and switching sessions should feel automatic before you move on.

6

Actually understand Git

Play Learn Git Branching — it makes branches and merges visual instead of abstract.

7

Learn the agent loop

Open Editors & AI and practice the shared loop: describe, plan, edit, review the diff, verify, and commit. That loop is the center of AI-native development here.

8

Choose a development path

Open Development and pick one progressive route: Apple apps, React/Next/Node, Rust, Python, ML, or LLMs. Build one small thing before collecting more links.

9

Learn the Mac layer

Use macOS windows to practice app switching, tabs, full screen, Raycast window movement, Anybox link recall, and BetterDisplay resolution presets.

10

Practice deliberately

Use the Practice plan for daily reps and the Resource library for outside reading, videos, paid practice, and official docs.

11

Put it together

See how it all combines in Everyday workflow, then pick a small project and spend an hour in the terminal, editor, and tmux together. Real use is where it sticks — experiment freely; you can always reset with one command.
