The daily path

Groundwork Twelve.

84 days from zero to AI-native development habits. Twelve weeks, one focused hour on weekdays, one deep build every Saturday — with real progress you can point at, and an agent that starts as your tutor and ends as a delegate you supervise.

Groundwork Twelve combines pieces that are usually separate: from-zero tool fluency, a day-by-day schedule, hours-deep practice that builds muscle memory, honest progress measurement, and directing AI agents as a first-class skill. It sequences the pages already in this guide — each day tells you what to do, how long it takes, and where to read.

It's written in layers, like everything here. If you already know a week's material, take its skill check on Monday: pass it and skip ahead, and let the plan slow you down only where your fingers (not your understanding) need the reps.

#The weekly rhythm

Every weekday hour has the same four slots. The plan fills them in for each day below:

⟳ Drill 20mRepeat an earlier skill until it's automatic. Skills come back on a fading schedule — about two days after you learn them, then a week, then three weeks. The day entries name what to re-drill so you never have to plan it. Skills you use constantly (navigation, Git) stop needing drills because the work itself is the rep.
New 25mToday's new material — a section of a Groundwork page, hands on the keyboard while you read.
Do 10mA small concrete task using the new thing, unassisted.
Log 5mOne honest line in log.md, committed. The commit is the day.
SaturdayThe deep build: 2–4 hours, blank canvas, using only skills from the weeks so far. This is not extra practice — it is the week's assessment, and it produces an artifact that lives in your repo.
SundayRest, or catch up on a missed day, plus a three-line weekly review in the log.

Life happens, so every day has three modes. The streak survives as long as you do the minimum:

Full day60–75 minutes — the plan as written.
Minimum day15 minutes — one drill, one log line, one commit. That's a real day.
Catch-up day90–120 minutes — yesterday's material plus today's. Never stack more than one missed day; if you're two behind, skip the older one and keep moving.
Recruit one buddy

People finish paced programs together far more often than alone. One friend on the same schedule — or even one person who just reads your log once a week — changes the odds more than any tool on this site.

#Play your way through it

If games are what keep you coming back, real ones exist for most of these skills — and they're legitimate substitutes for the ⟳ drill slot (never for the Saturday build). The day entries point at them where they fit:

bashcrawlA dungeon crawler that is a folder tree: explore with cd, examine with ls, read scrolls with cat, fight monsters with scripts. The gentlest possible start — Week 1.
OverTheWire: BanditThe classic. SSH into level 0, find the password to level 1, repeat for 30+ levels of escalating shell cleverness. Weeks 2–6 and beyond.
The Command Line Murder MysterySolve a murder with grep, pipes, and wits — a text-adventure whodunit made of files. The Week 2 Saturday build has a game mode built on it.
cmdchallengeOne-line shell puzzles in the browser, instant feedback. Great minimum-day material.
Learn Git BranchingThe most-loved Git visualizer: watch the commit graph move as you type commands. Week 4's secret weapon for branches.
git-game & Oh My Git!A treasure hunt played inside a real repo with real commands, and an open-source game that visualizes Git's internals live. Weeks 4–6.
Vim Adventures & vimtutorZelda-meets-text-editing in the browser, and the 30-minute interactive tutorial already installed on your machine. Week 5.
SQLBoltInteractive SQL puzzles in the browser — not dressed as a game, but it plays like one. Week 8.
The gaps are real

Nobody has made a good game for tmux, or for directing agents. If those existed we'd list them; they may yet come from here — the agent itself makes a surprisingly good game master (ask it: "invent a five-level terminal treasure hunt for me in a scratch folder and grade my solutions").

#How progress is measured

Four layers, no gimmicks. No points, no badges, no timers — artifacts and observable skills.

1. The daily logYour log is a Git repository — creating it is Day 1. One line per day, committed daily. The streak is git log, so measuring your progress is itself Git practice. Push it to GitHub for public accountability if that motivates you.
2. Weekly skill checksEvery week ends with a short "can you do this without the cheat sheet?" list. Observable, not timed. If you can't, repeat the drills — that's information, not failure.
3. The agent as examinerOnce a week from Week 4 on, the agent quizzes you (prompt below). It reads your log, sees what you've been avoiding, and gives you real feedback — the thing self-paced learning usually lacks.
4. Saturday artifactsTwelve builds, one per week, all in your repo — ending with a capstone. By Day 84 your repo is a portfolio of proof, not a certificate.

The log template

Day 1 creates ~/groundwork-log. Start log.md with this and add one line each day:

log.md
# Groundwork Twelve — my log

Day 1 (2026-07-07): Created this repo. pwd/ls/cd feel okay; rm scares me. 60m
Day 2 (2026-07-08): Navigation drill x5. Stopped reaching for the mouse twice. 60m
Day 3 (2026-07-09): Minimum day — one drill, this line. 15m

Each Sunday, add a short weekly review under a heading: what got easier, what still feels slow, and one thing to repeat next week. Honest beats impressive — the log is for you. (Restarting on a new machine later? new-twelve-log scaffolds the repo with these templates in place.)

The examiner prompt

Once a week (each Friday from Week 4 on), start your agent in the log repo and paste this — it also ships in the repo scaffold as prompts/examiner.md:

agent examiner
Read log.md and this week's files. You are my examiner, not my helper.
Give me five small tasks that test this week's skills, one at a time,
and wait for me to do each in my terminal and report back.
Don't show me how. After the five, tell me: what am I avoiding,
what should I repeat next week, and one thing I did well.

#Agent rules by phase

The agent is present from Day 1 — but what it's allowed to do grows as your fundamentals harden. This ladder is the heart of the plan: an agent that does the work for you in Week 2 quietly robs you of the practice that makes Week 10 possible. Each day below has an Agent: line applying these rules; state your phase when you start a session.

Weeks 1–3
Explainer		Allowed: explain commands and concepts, quiz you, point out mistakes after you try, generate practice drills.
Not allowed: write commands for you to copy blindly, edit files for you, solve a drill before you've attempted it.
Weeks 4–6
Reviewer		Allowed: everything above, plus review your commits and diffs, critique your scripts, give you a recovery plan when you're stuck.
Not allowed: make the commits, fix the script itself, or run recovery commands you don't understand yet.
Weeks 7–9
Bounded pair		Allowed: everything above, plus implement small pieces after you've written the spec or failing test, and suggest edge cases you missed.
Not allowed: choose what to build, or merge anything you haven't read line by line.
Weeks 10–12
Delegate		Allowed: own bounded tasks end to end — plan, edit, run checks, report back with a diff and verification notes.
Not allowed: skip the plan, skip verification, or commit. You own the merge — that never changes.
Why restrict a tool you're here to learn?

Because judgment is the skill. An agent can produce working output on Day 1, but the ability to direct, review, and verify that output only grows out of hands-on fluency. You climb the ladder as fast as your checks pass — the restriction is scaffolding, not punishment.

#The arc

Weeks 1–3Machine fluency: keyboard, terminal, files, the modern shell tools, tmux. Agent explains and quizzes only.
Weeks 4–6Craft: Git for real, your editor, pipes and scripts, publishing to GitHub. Agent becomes your reviewer.
Weeks 7–9Programming, data, and the web: code by hand, then SQL and document stores, then HTTP — the three things every real app is made of. Agent becomes a bounded pair.
Weeks 10–12Direction at scale: narrow agent tasks, then specs, plans, verification, and judgment across whole repos. Capstone.

#Week 1 — Keyboard, terminal, and your log

Reading: Getting started. Everything happens in the terminal from the first minute. Agent phase: explainer.

New: open Ghostty; learn pwd, ls, cd, mkdir, touch by using each five times.

Agent: ask it to explain each command in one sentence. Don't let it run them.
Day 2 · TueMove without the mouse.
⟳ Drill: Day 1's five commands, from memory.
New: line editing — Tab completion, CtrlA/E, AltB/F, CtrlL.
Do: the walk-the-tree drill from Getting started, five rounds.

Agent: ask for a ten-question quiz on today's keys.
Day 3 · WedFiles and folders.
⟳ Drill: walk-the-tree (Day 2), two rounds.
New: mv, cp, rm -i in a scratch folder under /tmp.
Do: delete the wrong file on purpose, feel the stomach drop, recreate it — that feeling is why Week 4 exists. Game option: spend the Do slot in bashcrawl instead — it drills exactly today's commands.
Log: commit it.
Agent: after your attempt, ask it to point out anything dangerous in how you used rm.
Day 4 · ThuShell comfort.
⟳ Drill: files drill (Day 3), fresh scratch folder.
New: history with CtrlR; --help and man; the survival kit (CtrlC, q, CtrlD).
Do: re-run yesterday's drill typing nothing you've typed before — history only.

Day 5 · Fritmux, first contact.

Log: check results, honestly.
Agent: ask it to explain what a session actually is, like you're smart but new.
Saturday build · Day 6 (2–3h)

Your workspace, repeatable. From a fresh terminal, build the setup you'll use every day: a tmux session with a pane for work, a pane for notes, your log open. Then write workspace.md — the exact steps to open it from cold — and commit it. Blank canvas: no copying from this page, only from your own history and notes.

Week 1 check — without the cheat sheet: navigate anywhere and back; create, move, and delete a file; recall a command with CtrlR; start, split, detach, and reattach tmux; seven commits in your log.
