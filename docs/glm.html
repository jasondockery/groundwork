Open-weights models

Open models are the option you own.

GLM is the leading open-weights model family for coding and agentic work, and it stands here for a whole category: models whose weights you can download, run locally, or point any agent at. Knowing how to use one keeps every vendor honest.

When you'll want this You want a capable model without a subscription, need work to stay on your own machine, or want to understand what "open weights" buys you as leverage.

#What "open weights" means, and why you care

A frontier lab like Anthropic or OpenAI serves its models only through its own apps and APIs. An open-weights model publishes the model file itself — you can download it, run it on hardware you control, and no one can change the price, the behavior, or the terms underneath you. For an agentic engineer that is not ideology; it is leverage: the skill of pointing your tools at any model means no single vendor owns your workflow.

#Meet GLM

GLM is the flagship open-weights family from Z.ai, built for agentic engineering, and the standard-bearer for the category: GLM-5.2 (released mid-2026) leads open models on the coding and terminal benchmarks that matter for agent work while competing with closed frontier models. You can use it three ways, in rising order of ownership:

Hosted API & coding plansZ.ai serves GLM through an API and subscription coding plans, priced well under the closed frontier labs. Several coding agents can be pointed at it as their backing model.
Hosted by othersBecause the weights are open, third-party providers serve GLM too — you can pick a host on price, region, or privacy terms.
Run it yourselfDownload the weights and run them on your own hardware. The full model needs serious memory; smaller variants and quantized builds run on a well-equipped Mac.

#Local models on this machine

Groundwork installs Ollama, a local model runtime that downloads and serves open models on your Mac at localhost:11434. It is the shortest path to a model that works with the network cable unplugged:

ollama run <model>Download (first run) and chat with a model in the terminal.
ollama psShow what's loaded and how much memory it holds — the local equivalent of a usage gauge.
ollama lsList the models you've downloaded and what they cost in disk.

The budget here isn't tokens or dollars — it's your machine's memory and patience. A model that fits comfortably answers fast; one that barely fits crawls and starves everything else. Know your AI budget covers reading that gauge.

#Honest tradeoffs

Local and open models are not a free replacement for the frontier: the closed leaders are still ahead on the hardest agentic work, and a model small enough to run comfortably on a laptop is well behind one served from a datacenter. The point is options — knowing what the open tier can do, what it costs to run, and when it's the right tool — so your choice of the frontier is a choice, not a dependency.
